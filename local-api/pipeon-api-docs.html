01 Arquitetura e bancos

O servidor opera com dois bancos MongoDB completamente separados — um para dados internos do Pipeon, outro para os sistemas-alvo.

Banco Pipeon

Armazena dados internos do sistema: usuarios, logs, agendamentos, backups e configuracoes. Configurado via PIPEON_MONGO_URL no arquivo .env. Nunca recebe dados de avaliacao.

Banco dos sistemas-alvo

Armazena dados de avaliacao (evaluations, notices, etc.). A connection string vem da sessao que o usuario seleciona no login — pode ser local ou remota.

Separacao obrigatoria Nenhuma colecao Pipeon (pipeon_logs, pipeon_scheduled, etc.) deve existir no banco do sistema-alvo. Os endpoints /api/pipeon/* sempre gravam no banco Pipeon, independente da sessao ativa.

Colecoes no banco Pipeon

ColecaoConteudoGravada por
pipeon_usersUsuarios do sistema Pipeon/api/users e /api/auth/login
pipeon_scheduledProcedimentos agendados/api/pipeon/scheduled
pipeon_logsHistorico de execucoes/api/pipeon/logs
pipeon_auto_backupsBackups das avaliacoes/api/pipeon/backup/create e cron
pipeon_settingsPermissoes por cargo e config/api/pipeon/settings/:docId

Configuracao do banco Pipeon

# local-api/.env

# Connection string do MongoDB dedicado ao Pipeon
PIPEON_MONGO_URL=mongodb://localhost:27017/

# Nome do banco (padrao: pipeon)
# PIPEON_DB_NAME=pipeon

Saida ao iniciar o servidor

Pipeon Local API rodando em http://localhost:5000
Banco Pipeon  : mongodb://localhost:27017/pipeon
Banco padrao  : mongodb://localhost:27017/
Scheduler     : verificando procedimentos a cada minuto

02 Autenticacao

O login retorna um token JWT valido por 8 horas. Todos os demais endpoints exigem esse token no header Authorization.

Header obrigatorio Inclua em todas as requisicoes (exceto o login): Authorization: Bearer <token>
POST /api/auth/login Publico Autenticar e obter token JWT
pipeon · pipeon_users

Autentica o usuario pelo e-mail. O cadastro deve existir em pipeon_users. Retorna token JWT com email, name e role no payload.

CampoTipoObrigatorioDescricao
emailstringSimE-mail cadastrado em pipeon_users
 
// Requisicao
POST /api/auth/login
Content-Type: application/json

{
  "email": "usuario@empresa.com"
}

// Resposta 200
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

// Resposta 401 — e-mail nao encontrado
{ "message": "Usuario nao encontrado no banco local." }
Atencao A senha nao e validada neste modo de operacao. O acesso e controlado pelo cadastro em pipeon_users e pela role atribuida.

04 Agendamentos

Gerencia procedimentos agendados. O scheduler interno verifica pipeon_scheduled a cada minuto e executa automaticamente os procedimentos com horario vencido.

Scheduler automatico O cron interno roda a cada minuto. Procedimentos com status = "pending" e scheduledFor <= agora sao executados automaticamente — sem interacao do usuario.
GET /api/pipeon/scheduled Auth Listar todos os agendamentos (ate 200)
pipeon · pipeon_scheduled

Retorna documentos ordenados por scheduledFor decrescente. Maximo de 200 por requisicao.

pipeon · pipeon_scheduled

07 Backups

O servidor busca as avaliacoes diretamente no banco-alvo e salva no banco Pipeon — sem trafego de documentos via HTTP. Ambas as conexoes ocorrem no servidor.

POST /api/pipeon/backup/create Auth Gerar e salvar backup server-side
pipeon · pipeon_auto_backups
Funcionamento O servidor conecta simultaneamente ao banco-alvo (para ler as avaliacoes) e ao banco Pipeon (para salvar o backup). Chamado automaticamente durante execucoes manuais e pelo cron em execucoes automaticas.

Como localizar backups no MongoDB

// Abrir o mongosh e conectar ao banco Pipeon
use pipeon

// Por numero do chamado
db.pipeon_auto_backups.find({ ticket: "123456" })

// Por e-mail do avaliador
db.pipeon_auto_backups.find({ evaluatorEmail: "joao@empresa.com" })

// Por periodo
db.pipeon_auto_backups.find({
  createdAt: { $gte: "2026-06-01" }
}).sort({ createdAt: -1 })

08 Proxy MongoDB generico

Endpoint flexivel que roteia qualquer operacao MongoDB para o banco e colecao especificados no body. Usado pelo frontend para todas as operacoes nos dados de avaliacao.

Uso restrito a dados de avaliacao Este endpoint nao deve ser usado para colecoes Pipeon (pipeon_logs, pipeon_scheduled, etc.). Para metadados do sistema, use os endpoints /api/pipeon/* dedicados.
POST /api/mongodb/:action Auth Executar operacao MongoDB em qualquer banco
:actionEquivalente MongoDBCampos adicionais no body
findcollection.find(filter)filter, projection, limit
findOnecollection.findOne(filter)filter, projection
insertOnecollection.insertOne(document)document
updateOnecollection.updateOne(filter, update)filter, update, upsert
updateManycollection.updateMany(filter, update)filter, update
replaceOnecollection.replaceOne(filter, replacement)filter, replacement, upsert
aggregatecollection.aggregate(pipeline)pipeline
CampoObrigatorioDescricao
databaseSimNome do banco (ex: target-database)
collectionSimNome da colecao (ex: evaluations)
connectionStringNaoMongoDB URI (padrao: localhost:27017)

Filtros suportam {"$oid":"..."} para ObjectId e {"$date":"..."} para datas — o servidor resolve automaticamente via resolveExtendedJson.

// Buscar avaliacoes de um edital
POST /api/mongodb/find
{
  "database": "target-database",
  "collection": "evaluations",
  "filter": {
    "notice": { "$oid": "65a1b2c3d4e5f6a7b8c9d0e1" }
  },
  "limit": 100
}

// Resposta 200
{ "documents": [...] }

// Resetar status de avaliacoes (updateMany)
POST /api/mongodb/updateMany
{
  "database": "target-database",
  "collection": "evaluations",
  "filter": { "notice": { "$oid": "65a1..." } },
  "update": {
    "$set": { "status": "NOTSTARTED" },
    "$unset": { "evaluationAverage": "" }
  }
}

// Resposta 200
{ "matchedCount": 42, "modifiedCount": 42 }
